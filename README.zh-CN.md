# handoff

> 从"这个项目到底怎么回事"到"开干"——2 分钟内完成上下文交接。

[English Documentation / 英文文档](README.md)

---

`handoff` 是一个 **Claude Code Skill**，为任意项目自动生成和维护一份结构化的 `HANDOFF.md` 交接文档。每次新 session 打开项目时，Claude 读取这份文档即可快速对齐上下文，无需人工解释。

---

## 为什么需要它？

| 痛点 | handoff 怎么解决 |
|---|---|
| 每次开新 session 要重新解释项目背景 | 7 节硬结构，2 分钟读完，包含所有关键信息 |
| Claude 上下文窗口有限，口头解释太占空间 | 一份文件承载项目核心事实，不浪费 token |
| 团队协作时信息散落在不同人的脑子里 | 客观事实写到文件里，谁打开都能对齐 |
| 项目演进后旧信息过时 | 一键重跑，自动扫描最新状态，保留用户判断 |

---

## 核心设计

### 3 个原则

- **短** — 7 个固定章节，每节 ≤ 5 行，整份文档 2 分钟读完。
- **准** — 每条事实可追溯到 commit / issue / `file:line`。不写猜测。
- **可重跑** — 多次执行结果稳定。扫描代码 + 询问用户，不靠 AI "编"。

### 7 个固定章节

| # | 章节 | 内容 |
|---|---|---|
| 1 | **定位** | 项目是什么、给谁用、核心价值（1-2 句） |
| 2 | **技术栈** | 每层的选型及理由（表格） |
| 3 | **结构速览** | 目录树，深度 ≤ 2 |
| 4 | **数据模型** | ER 级粒度，实体 + 关系，不展开字段 |
| 5 | **关键决策** | 每条带 commit hash / issue 链接 / 代码坐标 |
| 6 | **启动 & 验证** | 启动命令 + 人工确认的"跑通标志" |
| 7 | **陷阱清单** | "不要做 X，因为 Y" 格式 |

---

## 工作流

```
扫描项目 → 询问用户（3 轮） → 生成文档 → Review diff → 写入文件
```

### 3 轮询问是核心

| 轮次 | 做什么 | 为什么 |
|---|---|---|
| **第 1 轮** | 确认 AI 扫描到的事实，补充定位/陷阱/跑通标志 | 机器能扫代码，但不知道"为什么这么写" |
| **第 2 轮** | 审视旧文档中的决策和陷阱，批量保留/修改/删除 | 项目演进后部分条目可能过时 |
| **第 3 轮** | 确认用户备注区内容 | 用户手动记录的内容永不自动覆盖 |

## 安装与使用

### 安装

```bash
npx skills add aIeXCai/handoff
```

### 使用

```
/handoff
```

或者在对话中直接说：「交接文档」「更新 HANDOFF」「写个 handoff」

### 何时使用

- 长 session 接近上下文上限，想"打包带走"项目状态
- 新 session 开始，想快速对齐项目现状
- 项目结构有较大变动，想刷新文档
- 新人加入项目，需要快速了解项目全貌

## 示例输出

生成的 `HANDOFF.md` 长这样：

```markdown
## 1. 定位
Grit 小程序 — 健身计划打卡工具，面向 CrossFit/Hyrox 爱好者。
核心价值：用结构化计划 + 主观感受记录替代零散的健身日志。

## 2. 技术栈
| 层 | 选型 | 理由 |
|---|---|---|
| 前端 | 微信原生 + WXML | 小程序生态首选 |
| 后端 | 微信云开发 | 免运维，CloudBase 集成 |
| 数据 | CloudBase 文档数据库 | 与云函数天然集成 |

## 3. 结构速览
src/
├── pages/          # 页面
├── components/     # 公共组件
├── utils/          # 工具函数
└── cloudfunctions/ # 云函数

## 4. 数据模型
- User → Checkin: 1 对多（用户有多次打卡记录）
- Plan → Exercise: 1 对多（计划包含多个动作）

## 5. 关键决策
- 嵌入单文档而非子集合存储 checkin → commit a3f2b1c
- 懒创建 checkin 记录 → src/utils/checkin.js:42

## 6. 启动 & 验证
- `npm run dev` → 微信开发者工具中看到首页计划列表即为跑通

## 7. 陷阱清单
- 不要手动改 checkin 的 date 字段 → CloudBase 触发器依赖该字段索引
- 不要在 onShow 里做重查询 → 小程序页面切换频繁触发

## 用户备注
<!-- handoff:manual-zone -->
（用户手动添加的备注，skill 永不自动覆盖）
<!-- /handoff:manual-zone -->
```

## 与其他工具的区别

| | handoff | README.md | CLAUDE.md | 口述 |
|---|---|---|---|---|
| **谁维护** | AI + 人工确认 | 人工 | 人工 | 人工 |
| **更新频率** | 一键重跑 | 偶尔更新 | 手动 | 每次都要说 |
| **内容范围** | 事实 + 决策 + 陷阱 | 项目介绍 | AI 行为指令 | 任意 |
| **可追溯性** | 每条可指到代码坐标 | 无强制 | 无强制 | 无 |
| **适用场景** | AI 辅助开发的交接 | 给人看的项目介绍 | 给 AI 的行为规则 | 一次性解释 |
